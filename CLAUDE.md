# 子供向けブラウザゲーム 仕様書（統合版 / Claude Code 用）

> このファイルがゲームの「正」の仕様です。リポジトリ直下に `CLAUDE.md` として置いてください。
> アート方向は **案A「クレヨン絵本」で確定**。最終的に **GitHub Pages で公開**します。

---

## 0. このプロジェクトの前提（重要）

- **完全な静的サイト**として作る（サーバ処理なし）。GitHub Pages でそのまま動くこと
- **ビルド不要 / 依存ライブラリなし**を基本にする（HTML + Canvas + バニラ JS）
- **パスは必ず相対パス**で書く（`/src/...` のような先頭スラッシュ絶対パス禁止）。
  GitHub Pages のプロジェクトサイトは `https://<user>.github.io/<repo>/` 配下に置かれるため、
  絶対パスだとリンク切れする
- **軽量・タッチ操作対応**（タブレットで滑らかに）。厳密な物理演算は不要

## 1. ゲーム概要

たくさんの小さな車が地面を左から右へ走る。プレイヤーは指（マウス/タッチ）で障害物を描き、
車はそれを乗り越えながら進む。一部の「工場車両」は障害物を削ってなめらかにしたり、
トンネルを掘ってバイパスを作ったりする。空中に描かれた障害物は落下して積み重なる。
やわらかい絵本のような世界観。

- 対象: 子供（読み書き不要、見て触って楽しめる）
- 端末: PC / タブレット。タッチ操作を主に想定

---

## 2. ゲームメカニクス

### 2.1 車（共通）
- 画面左から一定間隔でスポーンし、右へ走る
- 地面・障害物の「表面の高さ」に沿って走り、坂で車体が傾く
- 右端に到達したら、かわいい演出（ポンッと弾けて消える等）でゴール。一定確率で再スポーン
- ゆるい段差は乗り越える。急で高い段差（`MAX_CLIMB_STEP` 超）は越えられず、止まる／ぷるぷる震える

### 2.2 工場車両（特殊・たまに混ざる）
- **グラインダー車（削り車）**: 前方の急な段差の上部セルを少しずつ削り、なめらかにして通す。砂ぼこり＆キラキラ
- **トンネル車（掘削車）**: 障害物を正面から掘り、車高ぶんの横穴（トンネル）を開ける。穴の上のセルは落ちずに残り、トンネルの天井になる（=バイパス成立）

### 2.3 障害物（プレイヤーが描く）
- 指でなぞった軌跡を、小さなセル（ブロック）の集合として配置
- **空中に描くと落下**。地面や既存セルに着いたら、その位置で固まる（静止）
- 固まったセルは支えがなくても残る（=トンネルの天井が崩れない）
- 削り／掘削で消したセル以外は静止を保つ

### 2.4 全消去ボタン
- 画面**右上**に配置。押すとすべての障害物セルを消す（車は残す）
- アイコン中心（ほうき）＋ラベル「ぜんぶけす」。押したとき気持ちいいエフェクト（シュワッ）

## 3. 技術選定

| 項目 | 採用 | 理由 |
|---|---|---|
| 描画 | Canvas 2D + `requestAnimationFrame` | 軽量・依存なし・タブレットで高速・GitHub Pages で素直に動く |
| 物理 | **自作の軽量グリッドシミュレーション** | 落下・積み重ね・削り・トンネルを1モデルで統一でき最軽量 |
| 言語 | バニラ JavaScript（ES モジュール） | ビルド不要。https 配信の GitHub Pages では module がそのまま動く |
| フォント | Google Fonts（後述）をCDNから読み込み | 本体は軽いまま、かわいい書体を利用。失敗時はシステムフォントにフォールバック |

> 物理エンジン（Matter.js 等）は使わない。「削ってなめらか」「トンネルを掘る」は剛体物理だと
> 扱いが難しい。グリッド（セル）モデルなら、描く＝足す / 落とす＝重力 / 削る・掘る＝消す、で
> すべて統一でき、軽い。子供向けで「物理っぽい動き」で十分。

## 4. アートディレクション ＝ 案A「クレヨン絵本」（確定）

手描きのゆらゆらした線と、紙のような質感。あたたかく、寝る前の絵本のような安心感。
**視覚的な完成イメージは、別添 `design-options.html` の “A” パネル**（リポジトリの
`/reference/` に置いて参照すること）。

### 4.1 カラートークン（そのまま使う）

| 用途 | HEX |
|---|---|
| 背景（紙）base | `#fbf3e2` |
| 空 | `#cfeafa` |
| 丘 奥 | `#c3e6a8` |
| 丘 手前 | `#a9d98c` |
| 地面 | `#e9c98f` |
| 草 | `#9ed16f` |
| 車（3色ローテ） | `#f4a3a3` / `#97c4ef` / `#ffd887` |
| 工場車両 | `#f4b95e` |
| 障害物ブロック（4色ローテ） | `#c7a2e0` / `#f3b4cf` / `#bcd98e` / `#9fd6e6` |
| タイヤ | `#5a4029` ／ ホイール芯 `#f5e7cf` |
| 線・顔のインク | `#6b4f34` |

### 4.2 線・形・質感
- **輪郭線**: 茶色 `#6b4f34`、太さ約 2.5〜3px、`lineJoin/lineCap = round`
- **クレヨンのゆらぎ**: 輪郭の頂点に微小ジッター（±0.5〜1px）を加えて手描き感を出す。
  さらに同じ輪郭を“ほんの少しずらして2回”描くと、クレヨンの重ね塗り感が出る
- **角丸**: 半径 ≒ 7px（やわらかく）
- **紙の質感**: 起動時に一度だけ薄いノイズを生成して全面に重ねる（不透明度 6〜10% 程度）。
  base 色 `#fbf3e2` の上に薄ノイズ → 紙っぽさ。毎フレーム再生成しない（軽量化）
- **頬の赤み**: 車の顔に小さなピンク `#ff9bb0`（不透明度 ~0.7）

> **キャンバスでの再現メモ**: 上記の「ゆらぎ」「重ね塗り」「紙ノイズ」を自前の描画関数で実装する。
> 手描き線ライブラリ `rough.js` を使えば簡単に出せるが、軽さ優先なら自前ジッターを推奨。
> 採否は実装者判断でよい（使う場合も CDN から1ファイルのみ）。

### 4.3 キャラクター
- **普通の車**: 角丸ボディ＋運転席のふくらみ＋水色の窓＋丸タイヤ2つ＋顔（白目に茶の瞳・にっこり口・頬）
- **工場車両**: 同じ系統のボディに識別要素 ＝ 前方に回転ドリル（横向き三角＋白いハイライト線）、
  上にハザードランプ、車体に白い帯。色は `#f4b95e`
- 顔のインクは `#6b4f34`

### 4.4 ボタン（全消去）
- 右上。背景 `#fffaf0`、枠 `#6b4f34`（2px）、文字色 `#6b4f34`
- ラベル「ぜんぶけす」＋ ほうき/キラッ アイコン

### 4.5 フォント
- 日本語（UI・ボタン）: **Zen Maru Gothic**（丸ゴシック・かわいい・日本語対応）
- 英数の装飾: **Fredoka**
- 読み込み: `https://fonts.googleapis.com/css2?family=Fredoka:wght@500;600;700&family=Zen+Maru+Gothic:wght@500;700&display=swap`
- フォールバック: `'Zen Maru Gothic','Fredoka',system-ui,sans-serif`

## 5. コア・データモデル（グリッドシミュレーション）

### 5.1 グリッド
- 世界をセルサイズ `CELL`（12〜16px）の格子で表す
- セル状態: `EMPTY` / `STATIC`（固まった障害物）/ `FALLING`（落下中クラスタの一部）
- セルは色を持つ（4.1 の障害物4色をローテ）

### 5.2 クラスタ（落下のかたまり）
- 1ストロークで描いた連結セル群を1つの「クラスタ」とする
- 毎フレーム重力で下降。1つでも地面/STATIC に接触したらクラスタ全体を `STATIC` に確定
- 確定後は支えがなくても落ちない（簡略化）→ トンネルの天井が保てる
- 「積み重なり」は、確定済みの上に別クラスタが落ちて止まることで自然に成立

### 5.3 地面の高さ（車が走る面）
- 各列 x について「一番上の STATIC/地面セルの高さ」= `heightAt(x)`
- 車はこの高さに沿って走る。前後輪の高さ差で車体角度を決める

### 5.4 削り・掘削
- **削る**: 前方の段差が急なとき、上部セルを数個 `EMPTY` にして段差を下げる
- **掘る**: 正面・車高ぶんのセルを横方向に `EMPTY`。上の STATIC は残す（トンネル天井）

---

## 6. 車のロジック（簡易物理）

```
毎フレーム:
  x += speed
  groundY = heightAt(車の前方)
  目標Y = groundY - 車高/2
  step = heightAt(前輪) - heightAt(後輪)
  if step > MAX_CLIMB_STEP:
      普通車      : 停止/震える（詰まり）
      グラインダー : 前方上部を削って step を下げる → 進む
      トンネル車   : 正面に横穴を掘る → 進む
  else:
      Yを目標へなめらかに補間し、傾きを更新して進む
  右端到達 → ゴール演出 → 消滅（or 再スポーン）
```

## 8. ファイル構成（静的・相対パス）

```
/index.html            … Canvas と起動（<script type="module" src="./src/main.js">）
/.nojekyll             … 空ファイル（GitHub Pages の Jekyll 処理を無効化）
/README.md             … 概要と公開手順
/src/config.js         … 調整用の定数（重力・セルサイズ・速度・しきい値・カラートークン）
/src/grid.js           … セルグリッド/クラスタ/重力/削り/掘削
/src/cars.js           … 車の種類・スポーン・地面追従・特殊行動
/src/input.js          … ポインタ/タッチ → セル配置（ブラシ）
/src/render.js         … Canvas 描画（背景・地面・障害物・車・クレヨン質感・エフェクト）
/src/ui.js             … 全消去ボタン等
/src/audio.js          … 効果音（任意・後回し可）
/reference/design-options.html … アートの完成イメージ（案A参照用）
/assets/               … 画像/音（必要になれば。相対パスで参照）
```

- すべて相対パス（`./src/...`, `./assets/...`）。先頭スラッシュ禁止
- 4.1 のカラートークンは `config.js` に集約し、各モジュールはそこを参照

## 9. GitHub Pages 公開手順

1. リポジトリを作成し、上記ファイル一式を **リポジトリのルート**に置いて push（`main` ブランチ）
2. ルートに空の `.nojekyll` を必ず置く
3. GitHub → **Settings → Pages** を開く
4. **Source = “Deploy from a branch”**、**Branch = `main` / `(root)`** を選んで Save
5. 数十秒後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される
6. 確認: サブパス配信でも崩れないこと（=絶対パスを使っていないか最終チェック）

### ローカル確認
- ES モジュールは `file://` 直開きでは動かない。簡易サーバで確認する:
  - `python3 -m http.server` → ブラウザで `http://localhost:8000/`
- スマホ/タブレット実機でタッチ操作と動作の軽さを必ず確認する

## 10. 実装ステップ（この順で段階的に作る）

各ステップで「動くもの」を作り、確認しながら進める。

1. **土台**: Canvas 全画面化（DPR対応）＋ゲームループ＋空・丘・地面（案Aの色・紙質感）
2. **お絵かき**: ポインタ/タッチで軌跡をセル配置（まだ落ちない）。クレヨン風の線
3. **重力**: 描いたクラスタが落下→地面/既存セルで固まる→積み重なり
4. **普通の車**: 左からスポーン、地面追従で右走行、急な壁で詰まる、顔つきで可愛く
5. **グラインダー車**: 詰まりを削ってなめらかにし通す（砂ぼこり・キラキラ）
6. **トンネル車**: 障害物に横穴を掘ってバイパスを作る
7. **全消去ボタン**（右上）＋UI＋シュワッ演出
8. **仕上げ**: ゴール演出、クレヨン質感の磨き込み、パララックス、効果音（任意）

## 11. 調整パラメータ（config.js 初期値の目安）

| 定数 | 役割 | 目安 |
|---|---|---|
| `CELL` | セルの大きさ(px) | 14 |
| `GRAVITY` | 落下速度 | 8 |
| `BRUSH_RADIUS` | 描画ブラシ半径(セル数) | 2 |
| `CAR_SPEED` | 車の速度 | 1.2 |
| `MAX_CLIMB_STEP` | 乗り越えられる段差(px) | CELL * 2 |
| `SPAWN_INTERVAL` | 車のスポーン間隔(ms) | 1500 |
| `FACTORY_RATIO` | 工場車両の出現割合 | 0.25 |
| `GRIND_RATE` | 1フレームで削るセル数 | 1〜2 |
| `MAX_CARS` | 同時に走る車の上限 | 24 |

実機で触って気持ちよく調整する前提。

## 12. パフォーマンス指針（軽量を守る）

- `CELL` は 12〜16px を維持（小さすぎるとセルが増えて重い）
- 更新は FALLING クラスタとその周辺の STATIC のみ。全セル走査しない
- 確定済み障害物はオフスクリーン Canvas にキャッシュし、変化時だけ再描画
- 紙ノイズ・テクスチャは起動時に1度だけ生成して使い回す
- 同時に走る車に上限（`MAX_CARS`）
- まず正しく動かし、その後プロファイルして重い所だけ最適化
